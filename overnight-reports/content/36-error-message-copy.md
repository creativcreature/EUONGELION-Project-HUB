# EUONGELION Error Message Copy

*User-friendly error messages maintaining the sacred, calm tone*

---

## Error Message Philosophy

Errors are frustrating, but they don't have to be jarring. EUONGELION error messages should:

1. **Acknowledge** what went wrong clearly
2. **Reassure** without being dismissive
3. **Guide** toward resolution
4. **Maintain** the calm, sacred aesthetic
5. **Never** blame the user

---

## Network & Connection Errors

### No Internet Connection

**Headline:**
> You're offline

**Body:**
> It seems you've lost connection. Check your internet and try again.
>
> If you've downloaded content for offline reading, you can still access it below.

**Button:** Try Again
**Link:** View Offline Content

---

### Slow/Unstable Connection

**Headline:**

**Body:**

>
> Give it a moment, then try again.

**Button:** Retry

---

### Request Timeout

**Headline:**
> This is taking too long

**Body:**
> The content isn't loading. This could be a temporary issue.

**Button:** Try Again

**Body:**

## Tone Guidelines

### Voice:
- **Honest:** Tell them what went wrong
- **Calm:** Don't create panic
- **Helpful:** Show a path forward
- **Humble:** Take responsibility when it's ours

### Sample Contrasts:

| Avoid | Prefer |
|-------|--------|
| "Error 500" | "Something went wrong" |
| "Invalid input!" | "Please check your entry" |
| "Failed to authenticate" | "That didn't match our records" |
| "Request failed" | "This didn't go through" |
| "Access denied" | "This content requires premium" |
| "Unknown error" | "We encountered an unexpected problem" |

---

*Voice note: Error messages are moments of friction. EUONGELION's response should be to reduce frustration, not add to it. Every error is an opportunity to demonstrate care and competence through calm, clear communication.*
